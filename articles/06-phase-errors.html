How to Detect and Correct Quantum

In the previous article, we introduced the quantum bit flip error and the

Can we use the same circuit for detecting

bit flip errors to detect phase errors?

After a phase-flip error on the middle qubit, the corrupted state

remains a valid codeword, that is, it still has components only along

|000⟩ and |111⟩, placing it squarely within the codespace. From the

perspective of the bit-flip detection circuit, nothing looks wrong. The

syndrome measurement returns 00 every time, falsely indicating no

error.

This reveals a fundamental blind spot: the syndrome outcome is the

same regardless of which qubit the phase error struck. The bit-flip

detection circuit has no way to distinguish a phase-corrupted state

from a clean one. We therefore need a different approach to catch

phase-flip errors.

In the {|+⟩, |−⟩} basis, X acts as a phase flip and Z acts as a bit flip, exactly

swapping the roles of the two error types. The Hadamard transform

exchanges bit flip and phase flip errors.

This is the key insight: if we work in the {|+⟩, |−⟩} basis, we can apply

the exact same parity-checking methodology used for bit flip errors

to detect phase flip errors instead.

· · ·

In this article and the previous one, we have seen how to

independently correct bit-flip and phase-flip errors using parity-

checking circuits. It is worth noting a key limitation: the bit-flip

detection circuit is blind to phase errors, and the phase-flip detection

circuit is also blind to bit errors.

In the next article, we will see how concatenating both circuits

makes it possible to detect and correct both bit and phase flip errors

simultaneously.
