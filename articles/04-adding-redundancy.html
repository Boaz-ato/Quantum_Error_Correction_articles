Quantum Error Correction

How Do We Add Redundancy?

Boaz Micah

We have identified three obstacles that make quantum error correction hard: the no-cloning theorem, the continuous nature of quantum errors, and the destructiveness of measurement. Here, we tackle the first of these head-on: how do you add redundancy to quantum information without being allowed to copy it?

The Problem with Copying

In the classical repetition code, redundancy was straightforward: copy the logical bit multiple times, and use majority voting to recover from errors. The quantum analogue of this would be a map of the form:

\[ |\psi\rangle|0\rangle \rightarrow |\psi\rangle|\psi\rangle \]

The no-cloning theorem tells us this is impossible. No physical operation can duplicate an unknown quantum state. So the most direct translation of the classical strategy is ruled out from the start.

But this doesn't mean redundancy is impossible. It just means we have to be more careful about what redundancy means in the quantum setting.

Redundancy Through Entanglement

In quantum information, redundancy is achieved not by copying, but by expanding the Hilbert space. Instead of duplicating the state, we entangle the qubit we want to protect with additional qubits, spreading the logical information across a larger system.

To make this concrete, let's consider adding redundancy to a single qubit using three physical qubits. The circuit used to add redundancy is given below. It produces the encoding using two CNOT gates, with the original qubit as the control and the two extra qubits, initialised to \(|0\rangle\), as targets:

The circuit takes the state \(|\psi\rangle = \alpha|0\rangle + \beta|1\rangle\) and produces:

\[ |\psi\rangle = \alpha\,|0\rangle + \beta\,|1\rangle \xrightarrow{\text{three-qubit encoder}} |\psi\rangle_L = \alpha\,|000\rangle + \beta\,|111\rangle = \alpha\,|0\rangle_L + \beta\,|1\rangle_L \]

where \(|000\rangle\) and \(|111\rangle\) are the logical codewords. It is important to be clear about what this state is not. It is not a cloned state. Cloning would give:

\[ |\psi\rangle \otimes |\psi\rangle \otimes |\psi\rangle = (\alpha|0\rangle + \beta|1\rangle) \otimes (\alpha|0\rangle + \beta|1\rangle) \otimes (\alpha|0\rangle + \beta|1\rangle) \]

which expands into eight terms with all combinations of 0s and 1s. The encoded state \(\alpha|000\rangle + \beta|111\rangle\) has only two terms: the three qubits are entangled, not independent copies. The quantum information has been distributed across the joint state of the system.

The Codespace

The encoding operation gives us a clean geometric picture of how redundancy enables error detection. Before encoding, the single qubit lives in a two-dimensional Hilbert space:

\[ |\psi\rangle \in \mathcal{H}_2 = \mathrm{span}\{|0\rangle,\, |1\rangle\} \]

After encoding with three physical qubits, the system lives in an eight-dimensional Hilbert space \(\mathcal{H}_8\). But the logical qubit doesn't occupy all of it. It is confined to a two-dimensional subspace:

\[ |\psi\rangle_L \in \mathcal{C} = \mathrm{span}\{|000\rangle,\, |111\rangle\} \subset \mathcal{H}_8 \]

This subspace \(\mathcal{C}\) is called the codespace. It is the corner of the full Hilbert space where valid, uncorrupted logical states live.

Now, suppose a bit-flip error occurs on the first qubit. The corrupted state becomes:

\[ X_1|\psi\rangle_L = \alpha|100\rangle + \beta|011\rangle \]

This state is no longer in the codespace \(\mathcal{C}\). It has been rotated into a different two-dimensional subspace:

\[ X_1|\psi\rangle_L \in \mathcal{F} = \mathrm{span}\{|100\rangle,\, |011\rangle\} \subset \mathcal{H}_8 \]

We call \(\mathcal{F}\) the error subspace. Crucially, \(\mathcal{C}\) and \(\mathcal{F}\) are mutually orthogonal: they share no overlap. An uncorrupted logical state lives in \(\mathcal{C}\); a state that has suffered a bit-flip on the first qubit lives in \(\mathcal{F}\); and these two possibilities are, in principle, perfectly distinguishable.

Because \(\mathcal{C}\) and \(\mathcal{F}\) are orthogonal, we can perform a projective measurement that determines which subspace the state occupies, without learning anything about the values of \(\alpha\) and \(\beta\).

Think of the quantum state as a vector living in an eight-dimensional space, spanned by the eight computational basis states: \(\{|000\rangle, |001\rangle, |010\rangle, |011\rangle, |100\rangle, |101\rangle, |110\rangle, |111\rangle\}\). When no error has occurred, the state lives entirely within the codespace — it has components only along the \(|000\rangle\) and \(|111\rangle\) axes. Projecting onto any other axis gives zero.

When an error occurs, the state is rotated out of the codespace. It now has a non-zero component along one of the other axes, and which axis that is tells us exactly what error occurred. We can detect this by performing a projective measurement that asks a simple question: which subspace does the state live in?

Notice that this measurement never reveals the values of \(\alpha\) and \(\beta\). It only tells us which error subspace the state has been rotated into. The actual quantum data we care about remains untouched throughout. As such, we can detect the error without ever touching the logical information itself.

This is how redundancy is added in quantum error correction. Next, we'll make it concrete: introducing the quantum bit-flip code, and working through exactly how projective measurements are performed.
